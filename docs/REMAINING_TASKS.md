## 1. صفحة سجل الصيانة (MaintenanceLog) — غير منفذة

| المطلوب | الحالة |
|---------|--------|
| 4 بطاقات ملخص | ❌ غير موجودة |
| Timeline لتغيير الزيت | ❌ غير موجود |
| تصدير السجل | ❌ غير موجود |
| عرض تفاصيل كل سجل | ❌ غير موجود |

**ملاحظة:** الصفحة الحالية "السجل" (History) تعرض **السجل التاريخي للمستشعرات** (رسم بياني خطي، فلاتر، إحصائيات). أما **سجل الصيانة** (MaintenanceLog) فهو صفحة منفصلة تعرض سجل تغييرات الزيت بشكل Timeline.

**الإجراء:** إنشاء `lib/pages/maintenance_log_page.dart` وإضافتها للتنقل في `root_layout.dart` (قد يتطلب إعادة ترتيب الشريط السفلي أو إضافة تبويبات).

## 2. ربط الإعدادات بالتطبيق — جزئي

| الإعداد | مخزن | مطبّق في التطبيق |
|--------|------|------------------|
| وحدة الحرارة (C/F) | ✅ | ❌ لوحة التحكم تعرض °C ثابت |
| فترة التحديث (1,3,5,10 ثواني) | ✅ | ❌ Dashboard يستخدم 3 ثواني ثابتة |
| الوضع الليلي | ✅ | ❌ MaterialApp لا يطبق dark theme |
| الإشعارات + التنبيهات الحرجة فقط | ✅ | ❌ لا يُستخدم في منطق التنبيهات |
| إعادة الاتصال التلقائي | ✅ | ❌ زر WiFi لا يفعل شيئاً |
| مدة حفظ البيانات (7–90 يوم) | ✅ | ❌ لا يُطبق على فلترة/حذف البيانات |

**الإجراء:** ربط `SettingsProvider` مع:
- `DashboardPage`: استخدام `updateIntervalSeconds` و `temperatureUnit`
- `main.dart`: استخدام `darkMode` في `themeMode`
- منطق التنبيهات والإشعارات (إن وُجد)

## 6. هيكل التنقل الحالي vs المطلوب

**الحالي:**
1. الرئيسية (Dashboard)
2. التنبيهات (Alerts)
3. السجل (History) — سجل المستشعرات
4. التقارير (Reports)
5. الإعدادات (Settings)

**المطلوب (وفق التوثيق):**
